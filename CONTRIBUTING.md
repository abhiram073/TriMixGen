# Contributing to TriMixGen

First off, thank you for considering contributing to TriMixGen! It's people like you that make TriMixGen a better tool for everyone.

## Where do I go from here?

If you've noticed a bug or have a feature request, make sure to check if there's already an [issue](https://github.com/abhiram073/TriMixGen/issues) for it. If not, feel free to open a new one!

## Setting up your environment

1. Fork the repo and create your branch from `main`.
2. Follow the setup instructions in the `README.md`.
3. If you've added code that should be tested, add tests.
4. Ensure the test suite passes (`pytest backend/tests/`).
5. Ensure your code is formatted (PEP8 for Python).

## Submitting a Pull Request

- Please fill out the PR template completely.
- Reference any open issues your PR resolves.
- Keep your PRs focused on a single issue/feature.

Thank you!
